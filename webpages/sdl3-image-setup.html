SDL3 Image Visual Studio Setup! (Windows)

We can use the SDL Image library to load different image formats. Such as .png, .jpg, .gif, and .bmp.

Requirement List

  1. git the commandline tool!
  2. Visual Studio and the Desktop Development with C++ workload. (This workload can be found in the Visual Studio Installer app Modify > Desktop development with C++)

Cloning SDL_image from Github

Lets clone the SDL Image Github Repository. If you need a refresher please check the Cloning SDL3 from Github section of our SDL3 setup guide.

Updating SDL_image's External Dependencies

In the ./SDL_image/external folder their are a bunch of empty directories. To download the external dependencies execute the SDL_image/external/Get-GitModules.ps1 script.

Building SDL_image

Open SDL_image/VisualC/SDL_image.sln and make sure you target the Win32 (x86) architecture. Build both the Release and Debug binaries.

Linking SDL_image to our Visual Studio Project

Open the project's property page, to do this right-click the project in the Solution Explorer and click the Properties tile. Make sure your configuration is set to "All Configurations". In the project's property page add SDL3_image.lib to Linker > Input > Additional Dependencies.

Check the image below to see how to setup the VC++ Directories. In the projects property page add the SDL image include to the Configuration Properties > VC++ Directories > Include Directories my include is "C:/Projects/SDL_image/include". After adding the include, change the configuration to target Release and add the SDL Image Release binaries (for me this is located in the "C:/Projects/SDL_image/VisualC/Win32/Release" and "C:/Projects/SDL_image/VisualC/Win32/Debug" folder).

The last step is to copy the SDL3_image.dll from the SDL_image Debug and Release output folder to our projects Debug and Release output folder. My SDL3_image.dll was in the "C:/Projects/SDL_image/VisualC/Win32/Debug" and "C:/Projects/SDL_image/VisualC/Win32/Release" folder. The example is shown below

Testing SDL_image

The last step is to confirm that SDL image is setup and working correctly on your machine. Fortunately test code is provided below:

#include <iostream>
#include <SDL3/SDL.h>
#include <SDL3/SDL_main.h> // only include this one in the source file with main()!
#include <SDL3_image/SDL_image.h>

int main(int argc, char* argv[])
{
 const int WIDTH = 252;
 const int HEIGHT = 272;
 SDL_Window* window = NULL;
 SDL_Renderer* renderer = NULL;

 SDL_Init(SDL_INIT_VIDEO);
 window = SDL_CreateWindow("Hello SDL", WIDTH, HEIGHT, 0);
 renderer = SDL_CreateRenderer(window, NULL, SDL_RENDERER_ACCELERATED | SDL_RENDERER_PRESENTVSYNC);

 if (IMG_Init(IMG_INIT_PNG | IMG_INIT_JPG) & (IMG_INIT_PNG | IMG_INIT_JPG)) {
  std::cout << "Image initialization success";
  SDL_Texture* myImage = IMG_LoadTexture(renderer, "C:\\Users\\sanzo\\OneDrive\\Pictures\\Camera Roll\\joker-cropped.jpg");

  for (int i = 0; i < 300; ++i)
  {
   SDL_RenderClear(renderer);
   SDL_RenderTexture(renderer, myImage, nullptr, nullptr);
   SDL_RenderPresent(renderer);
  }

  SDL_DestroyTexture(myImage);
  IMG_Quit();
 }

 SDL_DestroyRenderer(renderer);
 SDL_DestroyWindow(window);
 SDL_Quit();
 return 0;
}

If a window pops up with your image and then closes everything worked great. I would test this program for both Release and Debug to make sure everything is good! This is what my program ended up looking like.
